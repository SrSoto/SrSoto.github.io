Machine Learning / NLP
PyTorch Rebuild for 60% FP Reduction

Re-engineered the core name-matching NLP model, transitioning from a legacy TensorFlow implementation to a high-performance PyTorch architecture optimized for sub-second watchlists validation.

View Model Performance Metrics
  • Challenge: High False Positive rates (~8%) in legacy name-matching models leading to operational fatigue in manual reviews.
  • Solution: Rebuilt the model from scratch using PyTorch, implementing a more representative dataset specifically oriented to international sanctions lists.
  • Impact: Achieved a ~60% reduction in False Positives (from 8% to 3% in testing, 5% to 1% in production) while maintaining zero false negatives.
